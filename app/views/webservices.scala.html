@(title: String, menu: String) @serber("Servicios Web", "Lógica de registro que permite la administración web de datos de registro desde el protocolo HTTP.", title) {

  1. La lógica de registro proporciona un mecanismo de administración que permite buscar, crear, obtener, actualizar y borrar datos de registro mediante unos servicios web.

  2. Los servicios web habilitan la posibilidad de intercambiar información con terceros (presentaciones de registro) y lo hacen utilizando alguno de los siguientes formatos:

    • XML: Este formato resulta especialmente útil para el intercambio de información genérica (documentos) entre dispositivos físicos o lógicos. Se trata de un formato muy extensible y versátil. En este caso, éste se utilizará para documentar el esquema de la ISO 2146:2010.

    • JSON: Este formato resulta especialmente útil para el intercambio de información específica (datos) entre la interfaz web y el servidor web. Se trata de un formato muy seguro e inteligible. En este caso, éste se utilizará para el intercambio información con el modelo de datos.

  3. Los servicios web son RESTful, es decir, ligeros y estándares. Unos servicios web RESTful permiten que la lógica de registro ocurra sobre el protocolo HTTP de forma óptima.

    API de servicios web RESTful para la lógica de registro:

    • (GET) /api/registry-object
    • (POST) /api/registry-object/:id
    • (GET) /api/registry-object/:id
    • (PUT) /api/registry-object
    • (DELETE) /api/registry-object/:id
    • …
