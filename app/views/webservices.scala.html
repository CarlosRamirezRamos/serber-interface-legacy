  1. Los servicios RESTful son ligeros y estándares, mediante un mínimo tráfico pemite la obtención de datos vía HTTP. Los servicios admitirán diferentes formatos o selrializaciones, JSON y XML.

    • JSON: Especialmente útil para el uso por interfaz Web y su consumo desde clientes Web. Es muy utilizado incluso para bases de datos no relacionales como MongoDB.
    • XML: Especialmente útil para el intercambio entre máquinas o aplicaciones. En este caso el schema sería el de la ISO.

  2. Un claro uso de los servicios Web en JSON es la propia interfaz Web con la que se va a dotar SERBER que los usa para intercambiar datos con el modelo.

  3. API:

    • (POST) /api/registry-object/:id
    • (GET) /api/registry-object/:id
    • (PUT) /api/registry-object
    • (DELETE) /api/registry-object/:id
    • ...
