@(title: String, menu: String) @serber("Servicios Web", "Lógicas de registro que permiten la administración, consulta y modificación web de los datos de registro desde el protocolo HTTP.", title) {

  1. Los servicios web son RESTful, es decir, ligeros y estándares. Unos servicios web RESTful permiten que las lógicas de negocio ocurran sobre el protocolo HTTP de forma óptima.

  2. Los servicios web RESTful suelen intercambiar información utilizando los siguientes formatos:

    • XML: Este formato resulta especialmente útil para el intercambio de información genérica (documentos) entre dispositivos físicos o lógicos. Se trata de un formato muy extensible y versátil. En este caso, éste se utilizará para documentar el esquema de la ISO 2146:2010.

    • JSON: Este formato resulta especialmente útil para el intercambio de información específica (datos) entre la interfaz web y el servidor web. Se trata de un formato muy seguro e inteligible. En este caso, éste se utilizará para el intercambio información con el modelo de datos.

  3. Un API de servicios web RESTful permite el intercambio de información tanto con el origen de datos (datos de registro), como con terceros (presentaciones de registro).

    API de servicios web RESTful:

    • (GET) /api/registry-object
    • (POST) /api/registry-object/:id
    • (GET) /api/registry-object/:id
    • (PUT) /api/registry-object
    • (DELETE) /api/registry-object/:id
    • …
