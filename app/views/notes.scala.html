  1. Tareas sobresalientes ya realizadas

    • Síntesis de un modelo de datos basado en el esquema de la ISO 2146:2010.

    • Construcción de un servicio de registro dotado de un API de servicios web RESTful que permite la obtención (modo lectura) de datos de registro encapsulados en el modelo de datos, vía web y utilizando el formato JSON.

    • Construcción de una aplicación "prueba de concepto" del servicio de registro dotada de una interfaz web responsive de búsqueda que utilice el API de servicios web RESTful para la obtención (modo lectura) de datos de registro encapsulados en el modelo de datos, vía web y utilizando el formato JSON.

    • Validación del modelo de datos mediante la carga de un lote de datos de registro procedentes de ALZIRA/DIANA en la aplicación "prueba de concepto" del servicio de registro.

  2. Tareas formales pendientes por realizar

    • Extensión de la aplicación "prueba de concepto" del servicio de registro de manera que se pueda utilizar el API de servicios web RESTful para la búsqueda, creación, actualización y borrado (modo escritura) de datos de registro encapsulados en el modelo de datos, vía web y utilizando JSON.

    • Elaboración de un entregable documentado.

  3. Conclusiones

    • El modelo de datos basado en el esquema de la ISO 2146:2010 ha demostrado ser perfecto para encapsular datos de registro de diferente origen y naturaleza.

    • Se observa la siguiente cadena de implicaciones: Si no se analiza cuidadosamente el origen de datos, esto es, los objetos de registro y en especial controlando sus relaciones, entonces no será posible establecer una correcta correlación entre el origen de datos y el esquema de la ISO 2146:2010, y, por ende, tampoco se le podrá sacar el máximo partido (vocabulario estándar y controlado, identificadores únicos, abierto en número y tipo de datos de registro, manejo de históricos, etc).
