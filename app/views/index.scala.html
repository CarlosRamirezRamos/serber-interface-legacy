Servicio de Registro de Bibliotecas y Entidades Relacionadas

Registro estandarizado de datos de bibliotecas y entidades relacionadas, españolas e internacionales, compilado con objeto de dar servicio a la comunidad cultural, al entorno bibliotecario, a la gestión pública y a los ciudadanos.

DEFINICIÓN ISO 2146:2010
Registro: colección de objetos registrados compilada para dar soporte a la actividad de una determinada comunidad.

Características del proyecto SERBER

  1. Interfaz Web responsive

    Presentación de registro que permite la gestión vía web de datos de registro encapsulados en un novedoso modelo de datos. Esta interfaz web es responsive, es decir, funciona de forma óptima sobre cualquier dispositivo. Leer más…

  2. Servicios Web RESTful

    Lógica de registro que permite la administración vía web de datos de registro encapsulados en un novedoso modelo de datos. Estos servicios web son RESTful, es decir, funcionan de forma óptima sobre el protocolo HTTP. Leer más…

  3. Modelo de Datos vanguardista

    Datos de registro encapsulados en un modelo de datos versátil y potente que sea recreable, mediante la ejecución de un script SQL, en cualquiera de los mecanismos de almacenamiento disponibles. La base de datos del proyecto piloto funciona sobre MySQL. Leer más…

  4. Metadatos

    Identificación y selección de vocabularios adecuados para la correcta representación de los objetos de registro y sus relaciones. Leer más…

  5. Datos Enlazados

    Creación de un esquema RDF que permita intercambiar datos de registro como datos enlazados SERBER. Leer más…

  6. Free & Open Source Software

    Utilización de software libre y código abierto. Énfasis en la contribución y en la adopción facilitando tanto la colaboración de otros programadores como la intercambiabilidad de los distintos componentes. Leer más…

Objetivos del proyecto piloto

El proyecto piloto pretende demostrar algunas de las características del proyecto SERBER:

  • Construcción de una aplicación de búsqueda dotada de una interfaz web responsive que utilice un API de servicios web RESTful para la obtención (modo lectura) de datos de registro encapsulados en un novedoso modelo de datos . Utilización de software libre y código abierto . Buscar y obtener datos de registro en el proyecto piloto.

  • Construcción de un servicio de registro dotado de un API de servicios web RESTful que permita la obtención (modo lectura) de datos de registro encapsulados en un novedoso modelo de datos . Utilización de software libre y código abierto . Leer notas sobre el proyecto piloto.

  • Síntesis de un novedoso modelo de datos basado en el esquema de la ISO 2146:2010 que permita encapsular datos de registro procedentes de ALZIRA/DIANA .
