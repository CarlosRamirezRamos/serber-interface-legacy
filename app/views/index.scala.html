¡Bienvenido a SERBER|piloto!


Servicio de Registro de Bibliotecas y Entidades Relacionadas

Registro estandarizado de datos de bibliotecas y entidades relacionadas, españolas e internacionales, compilado con objeto de dar servicio a la comunidad cultural, al entorno bibliotecario, a la gestión pública y a los ciudadanos.

DEFINICIÓN ISO 2146:2010
Registro: colección de objetos registrados compilada para dar soporte a la actividad de una determinada comunidad.

Características del proyecto SERBER

  1. Modelo de Datos vanguardista
    Modelo de datos recreable mediante la ejecución de un script en cualquiera de los mecanismos de almacenamiento disponibles. La Base de datos del proyecto piloto funciona sobre MySQL. Ver más…
  2. Servicios Web RESTful
    Servicios que permiten la consulta y administración vía web de los datos guardados en el modelo de datos. Estos servicios obtendrían los datos bien en JSON o bien en XML. Ver más…
  3. Interfaz Web responsive
    Interfaz que permite la consulta en modo lectura y escritura vía web de los datos guardados en el modelo de datos. Esta interfaz facilitaría la visualización y depuración de los datos desde cualquier dispositivo. Ver más…
  4. Metadatos
    Elección de vocabularios adecuados para una correcta definición de los objetos de registro y sus relaciones. Ver más…
  5. Datos Enlazados
    Creación de un esquema RDF que permita intercambiar como dato enlazado SERBER. Ver más…
  6. Open Source/Free Software (Código Abierto/Software Libre)
    Utilización de herramientas de código abierto/software libre. Cambio de arquitectura, cambio de mecanismo de almacenamiento, diseño y arquitectura modular. Separación de conceptos. Ver más…

Descripción del proyecto piloto

El proyecto piloto pretende implementar algunas de las características del proyecto SERBER:

  • Construcción de un registro con un API de servicios web RESTful que permita la consulta en modo lectura de sus datos. Implementación de un modelo de datos basado en el esquema de la ISO. Utilización de herramientas de código abierto/software libre . Ir a Notas: proyecto SERBER.
  • Construcción de una aplicación con una interfaz web responsive que utilice el API para la búsqueda y consulta en modo lectura de los datos del registro. Implementación de un sistema de acceso análogo al de ALZIRA o DIANA. Utilización de herramientas de código abierto/software libre . Ir a Datos: objetos de registro.
