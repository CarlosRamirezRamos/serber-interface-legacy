Servicio de Registro de Bibliotecas y Entidades Relacionadas

Registro estandarizado de datos de bibliotecas y entidades relacionadas, españolas e internacionales, compilado con objeto de dar servicio a la comunidad cultural, al entorno bibliotecario, a la gestión pública y a los ciudadanos.

DEFINICIÓN ISO 2146:2010
Registro: colección de objetos registrados compilada para dar soporte a la actividad de una determinada comunidad.

Características del proyecto SERBER

  1. Modelo de Datos vanguardista
    Software de código abierto. Script de creación del modelo ejecutable en varios motores. Base de datos del piloto MySQL. Ver más…
  2. Servicios Web RESTful
    Servicios que permiten administrar y consultar los datos guardados en el modelo. Permiten obtener los datos en JSON y en XML. Ver más…
  3. Interfaz Web responsive
    SERBER tiene uso principal de registro y servicio centralizador para aplicaciones que utilicen datos del directorio Diana, Alzira. Sin embargo, dispone de una interfaz Web que permite consultar y leer los datos para su visualización. Ver más…
  4. Metadatos
    Elección de los vocabularios adecuados para una correcta definición de los objetos de registro y sus relaciones. Ver más…
  5. Linked Data (Datos Enlazados)
    Creación de esquema RDF que permita intercambiar como dato enlazado SERBER. Ver más…

Descripción del proyecto piloto

El proyecto piloto pretende implementar algunas de las características del proyecto SERBER:

  • Construir el registro con un API de servicios Web que permita la consulta, lectura, adición y actualización del datos del registro. Hacerlo con un nuevo modelo de datos basado en el schema de la ISO y usando herramientas de código abierto
  • Construir la aplicación con una interfaz Web que sea capaz de utilizar ese API para consultar y leer los datos del registro. Hacerlo usando herramientas de código abierto . Igual que lo podría hacer Diana, Alzira. Ir a Datos: objetos de registro.
